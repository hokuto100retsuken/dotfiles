# Global CLAUDE.md

## 基本方針
- 日本語で会話する
- 簡潔に、要点だけ伝える。冗長な説明は不要
- 作業はゴリゴリ進める。確認より実行を優先
- ただし見た目や設計方針が曖昧なときは、方向性を先に確認してから進める
- コードを書いたらすぐテストも確認する

## コミットメッセージ
- 日本語で書く（Conventional Commitsは不要、プロジェクトのCLAUDE.mdで指定がない限り）
- 何をしたかが一目でわかる簡潔な文

## コーディングスタイル
- シンプルで読みやすいコードを書く
- 過度な抽象化をしない
- 既存コードの慣習に合わせる

## クリップボード
- ドキュメントや調査結果を求められたら、最終成果物をクリップボードにコピーする
- macOSでは `pbcopy`、Linuxでは `xclip -selection clipboard` または `xsel --clipboard` を使う
- コピーする内容はMarkdown形式で、前置き不要、本文のみ
- コピー完了したら「クリップボードにコピーしました」と一言伝える

## 開発環境
- macOS (Apple Silicon) と Linux のデュアル環境
- Fish shell
- Docker Compose での開発が中心
- エディタ: VSCode
