現在のセッションの引き継ぎメモを保存する。次回セッションで「前回の続き」とすぐ言えるようにするためのもの。

## 手順

1. 保存先パスを決定する:
   - `~/.claude/projects/$(pwd | sed 's|/|-|g')/handoff.md`
   - ディレクトリがなければ作成する (`mkdir -p`)

2. 以下の情報を収集する:
   - ブランチ名: `git branch --show-current`
   - 未コミット変更: `git status --short` の結果
   - 最新コミット: `git log -3 --oneline`
   - 今セッションでやったこと: 会話の要点を数行で
   - 次やること / 保留事項: 今セッションで未完了のタスクや、次回に持ち越す判断事項

3. 以下の形式で上書き保存する:

```markdown
# Handoff: <YYYY-MM-DD HH:MM>   # handoff 保存時点の日時 (例: `date '+%Y-%m-%d %H:%M'`)

## ブランチ
<branch-name>

## 未コミット変更
<git status --short の出力、なければ "なし">

## 最新コミット
<git log -3 --oneline の出力>

## 今セッションでやったこと
- <箇条書き>

## 次やること / 保留事項
- <箇条書き、なければ "特になし">
```

4. 保存完了したら、保存先パスと要点を1-2行で伝える。

## 注意

- 上書き保存 (前回のhandoffは消える)
- 会話の要点は簡潔に。詳細はgit logで追える
- 機密情報 (APIキー等) は書かない
- git管理外のファイル・ディレクトリに書くので、Writeツールで直接書いて良い
