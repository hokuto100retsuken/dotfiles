現在のブランチの変更内容をレビューする。

## 対象範囲
- base ブランチからの差分（コミット済み + 未コミット両方）

## 手順
1. base ブランチを特定（`git symbolic-ref refs/remotes/origin/HEAD` → 無ければ master / main）
2. `git diff <base>...HEAD` と `git status` で差分を取得
3. 以下の観点でレビューする：
   - バグやロジックの誤り
   - セキュリティ上の問題
   - パフォーマンスの懸念
   - テストの網羅性
   - 命名やコードの読みやすさ
4. 問題があれば具体的な修正案を提示
5. 問題がなければ「LGTM」と伝える

## 出力形式
- 観点ごとに見出しを立てて箇条書き
- 各指摘に対象ファイル / 行番号を添える
- 重要度（blocker / nit）を明示
- 些細なスタイルの指摘（nit）は基本的に省略、重要な問題（blocker）だけ指摘する
