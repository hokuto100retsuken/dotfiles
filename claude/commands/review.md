コードレビューを実行する。引数あり（URL / PR番号）ならそのPRを、なければ現在のブランチ差分をレビューする。

## 引数

- `$ARGUMENTS` が空 → 現在ブランチの差分レビュー
- `$ARGUMENTS` に URL または PR番号が含まれる → 指定PR のレビュー

## 手順（引数なし: ブランチ差分レビュー）

1. base ブランチを特定（`git symbolic-ref refs/remotes/origin/HEAD` → 無ければ master / main）
2. `git diff <base>...HEAD` と `git status` で差分を取得
3. 以下の観点でレビュー：
   - バグやロジックの誤り
   - セキュリティ上の問題
   - パフォーマンスの懸念
   - テストの網羅性
   - 命名やコードの読みやすさ
4. 問題があれば具体的な修正案を提示、なければ「LGTM」と伝える

## 手順（引数あり: PR レビュー）

1. `gh pr view <URL or PR番号> --json title,body,files,url` でPR情報を取得
2. `gh pr diff <URL or PR番号>` で差分を取得
3. 以下の観点でレビュー：
   - バグやロジックの誤り
   - セキュリティ上の問題
   - パフォーマンスの懸念
   - テストの網羅性
   - 命名やコードの読みやすさ
4. 問題があれば具体的な修正案を提示、なければ「LGTM」と伝える

## 出力形式

- 観点ごとに見出しを立てて箇条書き
- 各指摘に対象ファイル / 行番号を添える
- 重要度（blocker / nit）を明示
- 些細なスタイルの指摘（nit）は基本的に省略、重要な問題（blocker）だけ指摘する
