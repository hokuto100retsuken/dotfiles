# 失敗学習ループ

## 記録するタイミング
以下が起きたら、セッション終了を待たずにその場でプロジェクトのmemoryに記録する:
- ユーザーから作業のやり方を訂正された
- PRレビューで指摘を受けて修正した
- 同じエラーで2回以上手戻りした

## 記録フォーマット
`~/.claude/projects/<project-hash>/memory/` に1失敗1ファイルで保存:

```
### [タイトル] (YYYY-MM-DD)
- **状況**: 何をしようとしたか
- **失敗**: 何が間違っていたか
- **正解**: 正しいアプローチ
```

PRレビュー由来の学びは `pr-review-lessons.md` にまとめて追記する。

## 昇格メカニズム
- 別プロジェクトで同じ失敗・同種の指摘が再発したら、`~/.claude/rules/`（dotfilesの `claude/rules/`）へ昇格を提案する
- 昇格時はプロジェクト固有の文脈を削ぎ、汎用ルールとして1〜3行に圧縮する
- 昇格したらmemory側のファイルは削除する（二重管理しない）
