# コードスタイル・品質

## 役割ごとの書き方

| 対象 | 書く内容 |
|------|----------|
| コード | How |
| テスト | What |
| コミット | Why |
| コメント | Why not |

コミット文面の具体的な形式（種類プレフィックス / 手順 / 例）は `commit-message` skill を参照。本 skill では方針のみ示す。

## 変数名

補助動詞を含める: `isLoading`, `hasError`, `canSubmit`

## 原則

簡潔・効率的、読みやすく保守しやすく、重複を避けてモジュール化
