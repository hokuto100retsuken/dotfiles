# devスキル

人間は「最初（要件・計画承認）」と「最後（PRの最終確認）」だけ。間は自走する。

## フェーズ

### 0. 前提確認
- `git remote -v` でホスト、`git status` で現在地を確認
- ベースブランチをユーザーに明示してから作業ブランチを切る（ブランチ間違い防止）

### 1. 質問（足りない情報を取りこぼさない）
- タスクを読み、不足している情報を**最大3つ**質問する
- 十分に明確ならこのフェーズはスキップしてよい

### 2. 計画（曖昧な依頼を手順書にする）
- 実装手順書を提示する: サブタスク分解 / 対象ファイル / 検証方法 / 並列可否
- **ユーザーの承認を得る。承認後はフェーズ7まで自走し、途中で確認を挟まない**

### 3. 並列実装
- 独立したサブタスク（互いの出力に依存せず、書き込みファイルが競合しない）を Agent ツールで2〜3並列に起動する
- 依存関係があるものは直列に。1つしかなければ並列化しない

### 4. クロスレビュー
- 実装に関与していない**別エージェント**に差分をレビューさせる
- 観点: 正しさ / 既存パターンとの整合 / テストの妥当性。指摘には重要度を付けさせる

### 5. 修正ループ
- 指摘を修正 → 再レビュー。**上限2周**
- 解消できなかった指摘は握りつぶさず最終報告に残す

### 6. 整え
- lint / 型チェック / テストを実行して全て通す
- 小さな単位でコミット（commit-message スキルに従い日本語で）

### 7. ドラフトPR作成
- PRテンプレートの有無を確認し、`gh pr create --draft` で作成
- 最終報告: PR URL / 計画との差分 / 残った指摘・懸念。**Ready化と最終確認は人間がやる**
