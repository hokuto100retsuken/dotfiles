# Explainスキル

指定されたコードやファイルの処理フローを解説する。

## 手順
1. 対象のコードを読む
2. 処理フローを時系列で整理
3. 関連するモジュール（モデル・サービス・ジョブ・他関数等、プロジェクトの語彙に合わせる）の関係を説明
4. 設計上の意図や注意点があれば補足

## 出力形式
- 簡潔な箇条書き（目安: 5〜15 項目程度）
- 必要に応じてシーケンス図（Mermaid記法）
- 重要な分岐やエラーハンドリングを強調
