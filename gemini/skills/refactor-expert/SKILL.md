# Refactor Expert

このスキルは、コードの品質を維持し、長期的な保守性を確保するためのガイダンスを提供します。

## リファクタリングの原則
- **小さく変更する**: 一度にすべてを変えず、機能ごとに小さなステップでリファクタリングすること。
- **副作用の防止**: 機能的な動作を変えないこと。リンク作成やツールの動作に影響が出ないか常に確認すること。
- **一貫性の維持**: プロジェクト内の既存のパターン（例：`setup-dotfiles.sh` の `create_symlink` の使い方）に合わせること。

## dotfiles における主なタスク
- **設定の分割**: `config.fish` や `init.lua` が肥大化した場合、適切なサブディレクトリ（`conf.d/` や `lua/my/plugins/`）に分割すること。
- **重複の排除**: 複数のセットアップスクリプトで共通する処理（ディレクトリ作成、バックアップ等）を共通関数にまとめること。
- **古い記法の更新**: Fish や Neovim の新しいバージョンで推奨されている、より簡潔な記法へ移行すること。

## プロセス
1. **現状分析**: リファクタリング対象のコードを精査し、改善点を列挙する。
2. **プラン作成**: どのような変更を行うか、具体的な手順をユーザーに提示する。
3. **実行**: `replace` や `write_file` を使用して慎重に変更を適用する。
4. **検証**: シンボリックリンクの有効性、構文チェック、期待される動作（プロンプトの表示等）を確認する。
